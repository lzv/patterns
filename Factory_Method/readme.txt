Предположим, есть несколько редакторов - классов, наследующих один и тот же интерфейс Editor. Каждый редактор работает с документом своего формата. Каждый формат текста - это отдельный класс, и все они наследуют интерфейс Document. 
Интерфейс Editor имеет метод создания документа, возвращающий объект типа Document. Этот метод может быть как чисто виртуальным, так и иметь реализацию по умолчанию. Наследники интерфейса могут переопределять этот метод, что бы он возвращал тот конкретный тип документов, с которым работает этот конкретный редактор.
При такой ситуации этот метод создания документов и будет паттерном Фабричный метод. 
Если конкретный редактор работает с несколькими типами документов, то нужный тип можно указать через параметр метода. Так же возможен вариант, когда имеется только один класс редактора, и метод создания документа - шаблонный. Параметр шаблона - конкретный тип документа, который нужно создать и вернуть.
